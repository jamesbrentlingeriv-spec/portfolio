Overview

What This Project Delivers

Bible Study Suite brings together the tasks most users juggle manually: reading scripture, capturing notes, organizing references, and returning to previous study sessions without losing context. The project was built to keep that full workflow inside one app.

Beyond basic reading, the app supports an iterative study rhythm where users gather insight over time, revisit notes, and refine understanding passage by passage. The experience prioritizes continuity and retention rather than one-off interactions.

Highlights

Key Features

  • Integrated reading and note capture reduces context switching and keeps interpretation close to source passages.
  • Session-friendly structure supports repeated study over days or weeks with clearer continuity between visits.
  • Navigation patterns are tuned for quick movement between chapters, sections, and saved thought threads.
  • The interface balances simplicity for casual users while still supporting deeper study routines.
  • Architecture is modular enough to expand with additional research tools and study aids over time.

Red-Letter Text

Quoted speech in Matthew, Mark, Luke, and John is highlighted in red — just like a red-letter Bible.

📝

Notes

Create, edit, and autosave notes. Sidebar list for fast recall. Verse references typed in notes are auto-detected and rendered in the context sidebar.

📜

Referenced Verses Sidebar

Automatically fetches verse text for any references found in your notes. Shows reference cards and lets you jump to them instantly.

🎙️

Per-Note Audio

Record audio in-browser using your microphone. Recordings attach to the current note. Playback, download, and delete — persisted in IndexedDB.

🎵

Per-Note Songs

Add a song title and lyrics to any study note. View, delete, or insert saved songs directly into note content.

🏗️ Architecture

App flow: index.html → apps.js → notes.js / audio.js / bible.js → bible-api.com / localStorage / IndexedDB

Entity Fields
NOTE id, title, content, translation, timestamp
SONG id, title, lyrics, createdAt, noteId
RECORDING id, noteId, timestamp, duration, blob (IndexedDB)

🔧 Tech Stack

File Role
index.html App layout and UI containers
style.css Custom styling
apps.js Main controller, chapter rendering, navigation, search
bible.js Bible metadata, regex parsing, API calls to bible-api.com
notes.js Notes management, autosave, verse detection, songs UI/data
audio.js Recorder, waveform visualizer, IndexedDB audio persistence
Vanilla JS IndexedDB localStorage bible-api.com GitHub Pages Web Audio API

🔒

Privacy

Microphone access only when recording. All notes and audio stored locally in your browser. Bible text fetched from bible-api.com only.

🚀 Run Locally & Deploy

💻

VS Code Live Server

Install the Live Server extension, right-click index.html → Open with Live Server → http://127.0.0.1:5500/

🐍

Python

cd biblestudy
python -m http.server 8000
→ http://localhost:8000

📄

GitHub Pages

Push to main → Settings → Pages → Deploy from branch → main / root. Live in 5–10 min.

🛠️ Troubleshooting

Issue Fix
Blank Bible panel Use HTTP server (Live Server), not file://
No microphone Grant permission when prompted
Recordings disappear Avoid incognito — IndexedDB is cleared in private mode
GitHub Pages slow 5–10 min deploy time — Ctrl+Shift+R to hard refresh
